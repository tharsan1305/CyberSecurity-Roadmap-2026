# 79 - Governance

## Overview
Security Governance is the framework of policies, processes, and controls that ensure an organization's information security strategy aligns with its business objectives and legal obligations.

---

## 1. Security Policy Development

### Types of Policies
| Type                    | Description                                        |
|-------------------------|----------------------------------------------------|
| Information Security Policy | Master policy — sets overall direction       |
| Acceptable Use Policy (AUP) | Rules for using company systems/internet     |
| Password Policy         | Complexity, length, rotation, MFA requirements     |
| Access Control Policy   | Who can access what and how                        |
| Incident Response Policy| How to report and handle security incidents        |
| Data Classification Policy| How to classify and handle sensitive data       |
| BYOD Policy             | Rules for personal devices used for work           |
| Remote Work Policy      | Security requirements for remote employees         |

### Policy Components
Every policy should include:
1. **Purpose** – Why does this policy exist?
2. **Scope** – Who/what does it apply to?
3. **Policy Statements** – The actual rules
4. **Roles & Responsibilities** – Who is accountable?
5. **Enforcement** – What happens if violated?
6. **Review Date** – When will it be reviewed?

## 2. Security Frameworks Overview

### NIST Cybersecurity Framework (CSF)
Five core functions:
- **Identify** – Asset management, risk assessment
- **Protect** – Access control, training, data security
- **Detect** – Monitoring, anomaly detection
- **Respond** – Incident response, communication
- **Recover** – Recovery planning, improvements

### ISO/IEC 27001
- International standard for Information Security Management Systems (ISMS)
- Requires: risk assessment, risk treatment plan, Statement of Applicability, internal audits

### CIS Controls v8
18 Critical Security Controls prioritized by impact:
1. Inventory of Enterprise Assets
2. Inventory of Software Assets
3. Data Protection
4. Secure Configuration
5. Account Management
6. Access Control Management
...and 12 more

### COBIT 2019
- Governance framework for IT management
- Focuses on value delivery, risk management, resource optimization

## 3. Roles & Responsibilities

| Role                  | Responsibility                                    |
|-----------------------|---------------------------------------------------|
| CISO                  | Overall security strategy and leadership          |
| Security Manager      | Day-to-day security operations management         |
| SOC Analyst           | Alert monitoring, triage, incident response       |
| Security Architect    | Design and maintain security architecture         |
| GRC Analyst           | Governance, risk, compliance management           |
| Data Owner            | Responsible for data classification and access    |
| Data Custodian        | Technical management of data storage/backup       |
| End Users             | Follow security policies, report incidents        |

## 4. Enforcement & Awareness Programs

### Policy Enforcement Methods
- **Technical controls**: Firewalls, DLP, MFA enforcement, endpoint policies
- **Administrative controls**: Employee reviews, disciplinary procedures
- **Physical controls**: Badge access, CCTV, clean desk policy

### Security Awareness Training
- Annual mandatory training for all employees
- Phishing simulation campaigns (e.g., KnowBe4, Proofpoint)
- New employee onboarding security briefing
- Regular security newsletters and tips
- Role-specific training (finance: BEC awareness; IT: secure coding)

### Measuring Awareness Effectiveness
- Phishing simulation click rates (target < 5%)
- Training completion rates (target 100%)
- Incident reports from employees (higher = better awareness)
