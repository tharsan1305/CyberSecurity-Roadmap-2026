# Day 42 - Lab: SDLC and Sprint Simulation (Practical Exercise)

## Objective
Apply SDLC and Agile/Sprint concepts to your own roadmap progress, using GitHub Projects (from Topic 12) as a practical sprint board.

## Task 1: Map This Roadmap onto SDLC Phases
For your CyberSecurity-Roadmap-2026 project itself, identify:
- Planning: when you researched and structured the 100-topic roadmap
- Design: the folder structure/README checklist design
- Development: actually writing notes/labs daily
- Testing: doing the hands-on labs to verify understanding
- Deployment: pushing to GitHub
- Maintenance: updating/revisiting topics as needed

## Task 2: Set Up a GitHub Projects Sprint Board
On your roadmap repo:
1. Create a Project Board (if not done in Topic 12)
2. Create columns: Backlog, This Sprint, In Progress, Done
3. Add the next 5-10 topics from your roadmap as Issues, placed in Backlog

## Task 3: Run a Mini "Sprint Planning" Session (Solo)
Pick 5 topics for your "next sprint" (e.g., next 5 days of roadmap study), move them to "This Sprint" column.

## Task 4: Daily Standup Habit (Self-Check)
For the next few days, before starting your study block, write 2-3 lines answering:
- What did I complete yesterday?
- What will I work on today?
- Any blockers (time, understanding, tools)?

You can log this directly as a comment on the relevant GitHub Issue.
